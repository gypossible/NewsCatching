# 舆情监测助手

这是一个基于公开 Minimax expert 配置复刻出来的舆情监测工作台，核心能力包括：

- 主题与关键词监测
- 热点追踪与情绪判断
- 风险评分与行动建议
- 新闻分析员 / 社媒监测员 / 报告总控 三段式输出

当前版本不依赖任何私有 API Key，默认通过公开新闻 RSS 抓取信号，并支持你手工粘贴微博、小红书、评论区、客服记录等补充线索。
现在也支持上传 Excel / CSV 附件，按行批量导入监测主题。

## 技术栈

- Next.js 16
- React 19
- TypeScript
- `fast-xml-parser` 用于解析 RSS
- `xlsx` 用于在浏览器端解析 Excel / CSV 附件

## 本地运行

```bash
npm install
npm run dev
```

打开 [http://localhost:3000](http://localhost:3000) 即可。

## 已实现功能

- 左侧监测任务配置面板
- 右侧多角色 Agent 工作台
- Google News RSS + Bing News RSS 公开信号抓取
- 手工补充信号融合
- Excel / CSV 批量主题导入
- 结构化快报、风险等级、情绪分布、来源分布、时间走势

## 使用建议

1. 在“监测主题”中输入品牌、人物、产品或事件名。
2. 在“关键词补充”里加入 3 到 6 个高相关词。
3. 如果你已经有微博评论、论坛帖子或客服工单，直接粘贴到“手工补充信号”。
4. 如果你有一批主题，直接上传 Excel / CSV，按行批量执行监测。
5. 点击“启动监测任务”或“批量监测”，即可生成快报。

批量导入推荐列名：

```text
主题 | 关键词 | 监测视角 | 时间窗 | 分析要求 | 补充信号
```

如果文件没有表头，系统会默认按前六列依次读取。未填写的列会沿用左侧当前配置。

手工补充信号建议格式：

```text
[微博] 用户吐槽新品断货
[小红书] 评论区对包装设计评价两极
[客服] 今日关于配送时效的咨询量增加
```

Excel / CSV 中的“监测视角”可填写：

```text
品牌监测 / 危机预警 / 传播复盘 / 竞品分析
```

“时间窗”可填写：

```text
24h / 7d / 30d
24小时 / 7天 / 30天
```
